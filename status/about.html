What this is

UptimeProof PoA periodically produces “exports” (snapshots derived from monitoring data) and anchors the current head export into DNS TXT at _poa.uptimeproof.io. Each export can include a _poa header that links to the previous export (anti-rollback chain).

Single public source

All public pages here read from one endpoint only: /poa/v1/verify. That endpoint is expected to include enough information to: (1) parse DNS anchor, (2) match a local/public export, (3) report chain status.

Expected verifier fields (typical)

Your JSON may evolve, but these are common keys the UI will try to read:
verdict / ok
overall result of verification
dns.ts / dns.sha256 / dns.file
DNS anchor values
matched_file / matched_sha256
matched export on disk/public bundle
head.file / head.sha256 / head.ts
chain head info
head.prev_file / head.prev_sha256
anti-rollback link
chain.ok
chain integrity status

Security & open-source

Publishing the proof format + verifier code is good for trust and reproducibility. It does not give attackers control over your service — as long as you never publish secrets.
  • Do not publish DNS API tokens, private keys, SSH keys, passwords, or internal backups.
  • Publish only “safe” code: verifier, public specs, and examples with placeholders.
  • Use .env.example files with fake values and add real secrets to .gitignore.
